Sobre o Projeto

Este sistema foi desenvolvido para realizar o gerenciamento de empresas e funcionários, permitindo criar, editar, listar e excluir registros com integração entre front-end e API REST.

O sistema foi projetado para suportar o relacionamento N:N entre empresas e funcionários, permitindo que um funcionário participe de várias empresas e cada empresa tenha múltiplos funcionários.

Todas as entidades passam por validações antes de serem salvas no banco. Para empresas, são exigidos nome e endereço, além de um CNPJ obrigatório com exatamente 14 dígitos e sem duplicações. Para funcionários, o login deve ser único e composto apenas por letras, números, ponto e underline; o CPF precisa ter 11 dígitos e não pode se repetir; o e-mail é validado e deve ser único; e a senha precisa ter no mínimo 6 caracteres. Todas as operações incluem tratamento consistente de erros e, no caso de funcionários, a senha é automaticamente protegida através de hashing. A api retorna o hash da senha, mas caso isso não for oportuno basta comentar a funçao que passa a hash em EmployeeController Para que ela não seja cadastrada já hasheada.

Tecnologias Utilizadas

O front-end utiliza Angular 21 com componentes standalone, rotas com lazy loading, uso de Services para comunicação com a API.

A aplicação conta com páginas de listagem, criação, edição e detalhes, permitindo visualização de funcionários com suas empresas e empresas com seus funcionários,com sincronização via HTTP.

No back-end, foi utilizada uma API REST em Laravel, seguindo o padrão MVC. Os controllers implementam CRUD completo, validações e respostas estruturadas.

A arquitetura do Laravel inclui controllers dedicados, models bem definidos e relacionamento N:N através da tabela pivot company_employee.

API REST e Validações

A API segue os padrões RESTful utilizando os verbos HTTP GET, POST, PUT e DELETE para manipulação de recursos.

O EmployeeController possui validações como login sem caracteres especiais, CPF com 11 dígitos, e regras de unicidade com ignore para atualizações. A senha recebe hash automaticamente para garantir segurança.

O CompanyController implementa validações de CNPJ com 14 dígitos, nome obrigatório, endereço obrigatório e sincronização de funcionários por meio de arrays IDs enviados pelo front-end.

A API utiliza eager loading para retornar todos os relacionamentos já associados, garantindo eficiência e reduzindo múltiplas consultas ao banco.

Serviços do Angular

Os Services do Angular centralizam toda a comunicação com a API. O CompanyService e o EmployeeService possuem métodos padronizados para criação, edição, remoção e listagem, retornando respostas organizadas em um formato uniforme.

A aplicação também conta com um método interno chamado wrap(), utilizado para padronizar erros e respostas HTTP, facilitando o tratamento de mensagens no front-end.

Rotas de navegação

As rotas da aplicação foram organizadas utilizando carregamento dinâmico via loadComponent. A navegação inicia pela rota /home, responsável por apresentar a página inicial do sistema. O módulo de funcionários inclui rotas específicas para listar todos os registros em /employees, criar novos funcionários em /employees/new e editar um funcionário existente por meio da rota dinâmica /employees/:id. O módulo de empresas segue a mesma arquitetura, com a listagem geral em /companies, o formulário de criação em /companies/new e a edição em /companies/:id. Além disso, há uma rota exclusiva para visualizar informações detalhadas de uma empresa, disponível em /companies/:id/info. Por fim, qualquer rota inválida é tratada pelo fallback **, que redireciona automaticamente o usuário para /home.
